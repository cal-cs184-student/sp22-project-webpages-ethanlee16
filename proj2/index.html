Overview

In this assignment, we implemented rendering techniques for smooth curves, shading, and upsampling as well as mesh editing operations like flipping and splitting edges. We found that the composition of primitive mesh editing operations using algorithms like loop subdivision can yield useful results, like a smooth interpolation of coarse meshes.

Part 1

de Casteljau's algorithm is a method to evaluate polynomials by dividing line segments at a ratio (t) to draw a smooth curve between a set of points. The algorithm recursively interpolates each successive pair of points using the scalar parameter t. At each step of the algorithm, it goes through each pair of points and finds the linear interpolation with t, which makes a new intermediate point. The algorithm runs recursively finding intermediate points until it finds the final intermediate point.

In our implementation of one recursive step, we loop through each pair of points and find the linear interpolation by finding the lerp in the x and y directions and adding it to a vector of intermediate points.

Part 3

To find the area-weighted vertex normals, our implementation loops through each face that touches the vertex (by traversing through each halfedge’s twin’s next halfedge). For each face, we find all the vertices in the face and use their corresponding halfedge’s edge lengths to calculate the triangle’s area. To find the triangle’s area-weighted normal, we find the unit normal of the triangle (using the face’s normal function) and divide it by the triangle’s area. We add this area-weighted normal to a running sum of weighted normals. Once the loop goes through every neighboring triangle, we return the normalized sum of weighted normals.

Part 4

We implemented the edge flip operation with several pointer reassignment operations. With reference to the mesh below:

Flipping the edge connecting B and C (“BC/CB”) to become an edge connecting A and D (“AD/DA”) involves modifying 6 halfedges:

  • 2 halfedges involving BC/CB: BC and CB. These become AD and DA by resetting their next halfedge to DC and AB and rooting them in A and D, respectively.
  • 2 halfedges whose next previously pointed at BC and CB (now AD and DA): AB and DC. These should now point at BD and CA, respectively.
  • 2 halfedges whose next should now point at AD and DA (previously BC and CB): BD and CA. These halfedges will point to AD and DA.

One implementation trick that we found very helpful was leaving inline comments next to the names of the halfedges we were modifying. Both the variable names and the inline comment names came from drawing out the image and labeling each halfedge.

Because the meaning of BC and CB were changing, it was hard to repeatedly reference the halfedge in other calls to setNeighbors. With an inline comment, it was easier to quickly determine the new meaning of a particular halfedge. VS Code even includes a tooltip with the comment contents for each reference on hover:

Part 5

We implemented the edge split operation with pointer reassignments and additions to the mesh. Firstly, we create the new elements of the mesh: 2 new faces (the 2 triangles that are created as a result of the split), 3 new edges, and one new vertex (the midpoint/point that the split occurs on).

Then, we create 6 new halfedges (2 for each new edge) to support the new elements. We set the halfedges up using the diagram below:

The new halfedges are labeled: AM, MA, DM, MD, CM, MC. We set the properties of each of these halfedges using the new elements we have created, in addition to some references to existing halfedges.

Finally, each of the new elements’ halfedges are set to one of the halfedges, as well as that of the existing triangles (which may have used either the previous CB or BC halfedges). We return the new vertex M with the halfedge pointing in the direction of the edge that was split.

Part 6

Our loop subdivision algorithm follows 4 steps to take the mesh and increase the number of triangles:

  1. For every vertex in the existing mesh, we calculate its next position using the formula (1 - n * u) * position + u * neighbor_position_sum,where n is the degree of the vertex and u is 3/16 if the degree is 3 and 3/(8n) otherwise.

  1. For every edge in the existing mesh, we calculate the next position of the vertices to be created from splits using the formula: 3/8 * (A + B) + 1/8 * (C + D), where A and B are the vertices along this edge and C and D are the vertices opposite this edge. We store the next position of the new vertex created from the split in this edge, since each edge will receive one new vertex as the result of a split.

  1. For every edge in the mesh, we split it if it is not an edge created from a split. We mark the new edges resulting from the split as created from a split. Using the vertex that results from a split operation, we set its new position to the one stored on this edge.

  1. For every edge in the mesh, we flip it if it is an edge created from a split and it connects an old vertex to a new vertex. After we visit this edge, we mark it as not created from a split for the next subdivision pass.

Finally, we copy in the position of each vertex from the “next position” we stored in it from the above steps. We mark the vertices as not created from a split for the next subdivision pass.
